"Hola, somos el grupo de Sebastián Valencia, Daniel Alzate y Santiago Valencia. 
En este video mostramos el Nivel 1 del proyecto: la automatización de lectura de facturas 
electrónicas usando N8n."

### [0:30 - 1:30] MOSTRAR EL FLUJO
- Abrir N8n con el workflow ya creado
- Mostrar los nodos de izquierda a derecha

"Nuestro flujo tiene 7 nodos:
1. Dos triggers: uno manual para pruebas y uno programado cada 5 minutos
2. El nodo IMAP que se conecta al buzón de facturacion@suprokom.com
3. Un Code Node que extrae los archivos ZIP de cada correo
4. Un condicional IF que filtra los correos que sí tienen adjuntos
5. Otro Code Node que descomprime el ZIP y parsea el XML de la factura
6. Google Sheets donde se registran los datos extraídos"

### [1:30 - 3:00] MOSTRAR CONFIGURACIÓN DE NODOS CLAVE
- Click en el nodo IMAP → mostrar configuración (host, puerto 993, SSL)
- Click en el Code Node de parseo → mostrar brevemente el código
- Click en Google Sheets → mostrar la hoja destino

"El nodo IMAP está configurado con SSL en el puerto 993. El Code Node 
usa JSZip y xml2js para parsear el formato UBL 2.1 colombiano.
Un detalle importante es que las facturas vienen en formato AttachedDocument,
con la factura real embebida dentro del XML wrapper."

### [3:00 - 5:00] EJECUCIÓN EN VIVO
- Click en "Execute Workflow" (disparador manual)
- Esperar a que corra
- Mostrar la salida de cada nodo (el flujo de datos verde)
- Ir a Google Sheets y mostrar los datos registrados

"Ahora ejecutamos el flujo manualmente. Podemos ver cómo pasa por cada nodo...
Los correos se leyeron, se extrajeron los ZIP, se parsearon los XML y aquí en 
Google Sheets ya tenemos registradas las 3 facturas con NIT, razón social, 
número, fecha y monto."

### [5:00 - 5:30] CIERRE
"El flujo se activa automáticamente cada 5 minutos. Esto elimina por completo
la necesidad de que una persona revise correo por correo."

### [0:30 - 1:30] EXPLICAR ARQUITECTURA
- Mostrar la celda 1 del notebook (diagrama de arquitectura)

"La arquitectura tiene un agente ReAct creado con LangGraph que coordina

### [1:30 - 2:30] MOSTRAR CÓDIGO CLAVE

crea con create_react_agent de langgraph. El prompt del sistema le indica
que debe procesar TODOS los correos y presentar un resumen."

### [2:30 - 4:30] EJECUCIÓN EN VIVO + CHAIN OF THOUGHT
- Ejecutar la celda del agente (celda "Disparador Manual")

"Ejecutamos el agente... pueden ver la cadena de pensamiento en tiempo real:

luego extrae los adjuntos de cada uno →
después parsea cada XML y extrae NIT, razón social, monto...

"Aquí tenemos el resultado: una tabla con las 3 facturas procesadas

## TIPS PARA GRABAR

1. **Resolución:** 1080p mínimo (en Colab: View → Zoom 90% para que se vea todo)
2. **Micrófono:** Usa auriculares con mic, no el mic del portátil
3. **Antes de grabar:** Ejecuta todo una vez para que esté en cache y no esperar
4. **No te equivoques:** Si te trabas, pausa y sigue. Puedes editar después
5. **Duración:** N8n=5min, LangChain=5min. No te extiendas innecesariamente
6. **Sube a Google Drive** como "Cualquier persona con el enlace puede ver"
